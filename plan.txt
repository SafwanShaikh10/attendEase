1. System Overview
This system manages student leave and OD applications through a structured 4-level approval workflow. It handles three types of leave, enforces semester limits, auto-generates Excel records, and integrates letter templates for formal submissions.

4.3  Resubmission
When a request is Rejected or marked Resubmission Requested:
The request goes back only to the person who rejected or requested resubmission.
A new request is created and linked via parent_request_id.
The old request is frozen — never deleted.
Student can update reason and upload new proof/letter.
The new request goes through the full workflow again from that level.
4.4  Cancellation
Allowed only when status = SUBMITTED.
Once any coordinator has acted, cancellation is disabled.
Record stays in the database — never deleted.
Class Coordinator is notified so they do not act on it.

6. Letter System
Each leave type has its own separate letter template. When submitting, the student chooses one of two paths:
6.1  Option A — Type Digitally
Blank template is shown on screen with the correct format for the selected leave type.
Student fills in all fields directly in the browser.
System auto-generates a PDF from the filled content on submission.
PDF is attached to the request automatically — coordinators can view or download it.
6.2  Option B — Write Physically
Blank template is shown on screen for the student to copy the format by hand.
Student can also download the blank template as a PDF to print.
Student writes the letter on paper, takes a photo or scan.
Uploads the image or scanned PDF (JPG / PNG / PDF accepted).

7.2  Access
Stored in Google Drive folder scoped per coordinator.
Viewable as a rendered table inside the web application.
Downloadable as .xlsx from the website.
One file per division per semester — new approvals append a new row.
Class Coordinator sees only their division. Year Coordinator sees their full year. Chairperson has read-only access to all.
8. Substitute Approver System
To prevent delays when a coordinator is absent, Admin can assign a substitute approver temporarily.
Admin assigns a substitute via the admin panel for Class Coord or Year Coord roles only.
Substitute gets temporary rights scoped to that exact role and expires_at timestamp.
After expiry, access is automatically deactivated.
Audit log records: Approved by [Substitute Name] on behalf of [Coordinator Name].
No substitute for Chairperson — they are the singular authority for Special OD.
